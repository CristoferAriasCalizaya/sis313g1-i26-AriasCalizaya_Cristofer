PRÁCTICA N2 - GIT, GITHUB Y HTML
1. ¿Qué es una rama (branch) en Git y para qué sirve?

Una rama en Git es una línea de desarrollo independiente dentro de un proyecto. Sirve para trabajar en nuevas funcionalidades o cambios sin afectar la rama principal, permitiendo probar y desarrollar de forma segura.

2. ¿Cómo se crea y se cambia entre ramas en Git?

Para crear una rama se usa el comando git branch nombre_rama. Para cambiar a esa rama se utiliza git checkout nombre_rama. También se puede usar git checkout -b nombre_rama para crear y cambiar en un solo paso.

3. ¿Cuál es la diferencia entre la rama main y otras ramas?

La rama main es la rama principal del proyecto donde se encuentra la versión estable del código. Las otras ramas se utilizan para desarrollar nuevas funcionalidades o hacer pruebas sin afectar directamente el código principal.

4. ¿Cómo se combinan ramas en Git (merge)?

Las ramas se combinan utilizando el comando git merge nombre_rama. Esto permite unir los cambios de una rama a otra, generalmente integrando el trabajo realizado en una rama secundaria a la rama principal.

5. ¿Qué problemas pueden ocurrir al trabajar con ramas (conflictos) y cómo se resuelven?

Los conflictos ocurren cuando dos ramas modifican la misma parte del código. Para resolverlos, se deben revisar los archivos en conflicto, elegir qué cambios conservar o combinarlos manualmente, y luego guardar y confirmar los cambios con git add y git commit.
